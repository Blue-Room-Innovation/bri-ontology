How to use the auto-generated TypeScript types and JSON Schemas

## Basic usage

When you modify an ontology and the types and schemas are updated, they must be synchronized with the package. This is where the module that is imported for validation and type usage lives. To do this, run:

```bash
npm run contracts:build
```

- This updates the types and JSON Schemas of the package and performs a build.

```bash
npm run contracts:test
```

- This runs the test module under `/examples` to validate that everything still works.
  If it fails, it means you have changed a version or broken something, and you will need to adapt the scripts of this module so that it behaves as expected.

## Usage example

You can see an example of usage at:
`examples\consumer-node-ts\src\index.ts`

## Advanced Features

### Property Aliasing

You can override default property names using `skos:altLabel`.

By default, the build system derives property names from the URI's **local name**.
For example, given the property `bri-recycling:wasteTreatmentActivity`:

- **URI**: `https://.../recycling.ttl#wasteTreatmentActivity`
- **Default JSON Key**: `wasteTreatmentActivity`

In some cases, you may want the keys in your JSON data (and consequently in your TypeScript interfaces) to differ from the ontology's local name.

You can use the **`skos:altLabel`** annotation property within your SHACL property shape to specify an alias. The build system detects this label and uses it as the key in all generated artifacts.

1. Ensure the `skos` prefix is defined in your SHACL file:

```turtle
@prefix skos: <http://www.w3.org/2004/02/skos/core#> .
```

2. Add `skos:altLabel` to your `sh:property` definition:

```turtle
  sh:property [
    sh:path bri-recycling:wasteTreatmentActivity ;
    skos:altLabel "customActivityLabel" ;  # <--- Define your alias here
    sh:datatype xsd:string ;
    sh:maxCount 1 ;
  ] ;
```

This will generate:

- **JSON-LD Context**: `"customActivityLabel": { "@id": "bri-recycling:wasteTreatmentActivity", ... }`
- **JSON Schema**: `"properties": { "customActivityLabel": { "type": "string" } }`
- **TypeScript**: `export interface ... { customActivityLabel?: string; }`
