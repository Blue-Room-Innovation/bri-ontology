# Contribuir

## Flujo propuesto
1. Crea una rama descriptiva.
2. Valida OWL y SHACL localmente (`getting-started/quick-validation.md`).
3. Actualiza documentación relacionada (conceptos, referencia o guías).
4. Abre un PR con motivación, ejemplos y resultados de validación.

## Checklist PR
- [ ] Cambios OWL razonados (ROBOT `reason`) sin errores.
- [ ] Validaciones SHACL pasan con ejemplos.
- [ ] Documentación actualizada y consistente.
