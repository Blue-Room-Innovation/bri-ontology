# 15. Roadmap

Prioridades propuestas (tras renombrado):
1. Migrar shapes legado a modelo Waste/MARPOL (nuevas NodeShapes específicas).
2. Completar shapes específicos MARPOL (cardinalidades residuo vs puertos).
3. Añadir validación de códigos contra esquemas SKOS (SPARQL ASK / SHACL `sh:in`).
4. Automatizar CI (ROBOT + pySHACL + reporte HTML).
5. Extender soporte para eventos EPCIS asociados a movimientos de residuos.
6. Crear taxonomías SKOS adicionales (estado manejo, método tratamiento).
7. Documentar patrones de extensión (nuevos residuos regulados futuros).
