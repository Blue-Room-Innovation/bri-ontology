# 19. Contribuir

Flujo:
1. Crear rama descriptiva.
2. Cambios en ontología/shapes + ejemplos.
3. Validar local (OWL→SHACL).
4. Actualizar documentación afectada.
5. Pull Request con motivación y resultados.

Checklist:
- [ ] OWL razonado sin errores.
- [ ] SHACL conforme (ejemplos actualizados).
- [ ] Docs editados (sección adecuada).
- [ ] Nuevas IRIs documentadas si externas.
